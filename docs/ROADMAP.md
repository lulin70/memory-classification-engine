## 核心原则

基于"轻量级个人可携带"的定位，团队达成以下共识：

### 必须坚持 (MUST)

1. **🪶 保持轻量** - 零外部依赖，本地优先
2. **🎒 保持可携带** - 数据完全可控，随时导出
3. **👤 专注个人** - 不做企业功能，不做云端服务
4. **🆓 保持免费** - 核心功能永久免费

### 明确不做 (WON'T)

1. ❌ 云端存储服务
2. ❌ 团队协作功能
3. ❌ 企业级权限管理
4. ❌ 移动端原生 App

### P0 - 必须完成

#### 1.1 增强 CLI 工具 ⭐⭐⭐⭐⭐

**负责人**: 独立开发者  
**时间**: 1 周  
**优先级**: P0 (所有角色一致同意)

**功能**:
```bash
# 基础命令
carrymem add "I prefer dark mode"
carrymem list
carrymem search "theme"
carrymem show <id>
carrymem delete <id>

# 导入导出
carrymem export backup.json
carrymem import backup.json
```

- **Community**: 这能快速吸引早期用户

## 风险与缓解

### 高风险

1. **用户采用率低**
   - **缓解**: 简化使用，展示价值，社区推广
   - **负责人**: PM + Community

2. **工具集成困难**
   - **缓解**: 优先支持开放性好的工具，提供 MCP 标准
   - **负责人**: Architect

### 中风险

1. **性能问题**
   - **缓解**: 性能基准测试，持续优化
   - **负责人**: Architect + QA

2. **安全漏洞**
   - **缓解**: 安全审计，及时修复
   - **负责人**: Security

### 低风险

1. **社区参与度低**
   - **缓解**: 降低贡献门槛，激励机制
   - **负责人**: Community
