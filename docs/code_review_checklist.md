## 1. 通用检查项

### 1.1 代码规范

- [ ] **命名规范**
  - [ ] 类名使用 UpperCamelCase
  - [ ] 函数/变量名使用 snake_case
  - [ ] 常量名使用 UPPER_SNAKE_CASE
  - [ ] 私有成员使用 _ 前缀

- [ ] **代码格式**
  - [ ] 使用 4 个空格缩进（禁止 Tab）
  - [ ] 单行不超过 120 字符
  - [ ] 操作符前后有空格
  - [ ] 函数间有空行分隔

- [ ] **导入规范**
  - [ ] 标准库导入在前
  - [ ] 第三方库导入其次
  - [ ] 本地模块导入最后
  - [ ] 禁止使用动态导入（`__import__`）

### 1.2 文档规范

- [ ] **文档字符串**
  - [ ] 所有公共类有 docstring
  - [ ] 所有公共函数有 docstring
  - [ ] 参数和返回值有类型注解
  - [ ] 复杂逻辑有行内注释

- [ ] **README 更新**
  - [ ] 新功能有使用说明
  - [ ] API 变更有更新记录
  - [ ] 依赖变更有说明

## 2. Python 专项检查

### 2.1 异常处理

- [ ] **禁止裸 except**
  ```python
  # ❌ 错误
  except:
      pass
  
  # ✅ 正确
  except (ValueError, TypeError):
      pass
  ```

- [ ] **异常类型具体化**
  - [ ] 数据解析用 `(ValueError, TypeError)`
  - [ ] 文件操作用 `(OSError, PermissionError)`
  - [ ] 网络请求用 `(socket.error, OSError)`
  - [ ] 数据库操作用 `sqlite3.Error`

- [ ] **异常处理完整性**
  - [ ] 所有可能抛异常的地方有 try-catch
  - [ ] 外部调用有超时控制
  - [ ] 失败有重试机制（如需要）
  - [ ] 错误有日志记录

### 2.2 配置管理

- [ ] **禁止硬编码**
  ```python
  # ❌ 错误
  timeout = 30
  
  # ✅ 正确
  from utils.constants import DEFAULT_TIMEOUT
  timeout = config.get('timeout', DEFAULT_TIMEOUT)
  ```

- [ ] **魔法数字提取**
  - [ ] 时间值提取为常量
  - [ ] 阈值提取为常量
  - [ ] 限制值提取为常量
  - [ ] 权重值提取为常量

### 2.3 线程安全

- [ ] **共享资源保护**
  - [ ] 多线程访问共享数据有加锁
  - [ ] SQLite 连接使用 thread-local
  - [ ] 连接池有线程安全设计

- [ ] **避免竞争条件**
  - [ ] 检查-操作是原子性的
  - [ ] 使用线程安全的数据结构

## 3. 架构检查

### 3.1 模块化

- [ ] **单一职责**
  - [ ] 每个类只有一个职责
  - [ ] 每个函数只做一件事
  - [ ] 类大小 < 500 行
  - [ ] 函数大小 < 50 行

- [ ] **依赖管理**
  - [ ] 无循环依赖
  - [ ] 依赖抽象接口
  - [ ] 避免跨层调用

### 3.2 设计模式

- [ ] **Coordinator 模式**
  - [ ] StorageCoordinator 管理存储
  - [ ] ClassificationPipeline 管理分类
  - [ ] 职责分离清晰

- [ ] **插件系统**
  - [ ] 插件接口定义清晰
  - [ ] 插件加载机制完善
  - [ ] 插件错误隔离

### 6.1 单元测试

- [ ] **覆盖率**

  - [ ] 覆盖率 > 80%
  - [ ] 边界条件有测试

- [ ] **测试质量**

### 6.2 集成测试

- [ ] **接口测试**
  - [ ] API 接口有测试
  - [ ] 数据库操作有测试
  - [ ] 外部依赖有 Mock

## 7. 审查流程

### 7.1 审查步骤

1. **自我审查**（提交前）
   - [ ] 对照清单逐项检查
   - [ ] 运行测试套件
   - [ ] 检查代码风格

2. **同伴审查**（提交后）
   - [ ] 架构师审查架构设计
   - [ ] 测试专家审查测试覆盖
   - [ ] 开发者审查代码实现

3. **最终审查**（合并前）
   - [ ] 所有评论已解决
   - [ ] CI/CD 通过
   - [ ] 文档已更新

### 7.2 审查标准

| 级别 | 标准 | 处理方式 |
|------|------|---------|
| **必须修复** | 影响功能/安全/性能 | 修复后才能合并 |
| **建议修复** | 影响代码质量 | 建议修复，可讨论 |
| **可选优化** | 代码风格问题 | 可选，不强求 |

## 8. 工具集成

### 8.1 推荐工具

| 工具 | 用途 | 配置 |
|------|------|------|
| pylint | 代码规范检查 | `.pylintrc` |
| flake8 | 代码风格检查 | `.flake8` |
| mypy | 类型检查 | `mypy.ini` |
| pytest | 测试框架 | `pytest.ini` |
| coverage | 覆盖率检查 | `.coveragerc` |

### 8.2 自动化检查

```bash
# 代码风格检查
flake8 src/

# 类型检查
mypy src/

# 运行测试
pytest tests/ --cov=src --cov-report=html

# 完整检查
./scripts/code_quality_check.sh
```
