# Phase 2 需求文档

## 1. 项目概述

本项目是记忆分类引擎（Memory Classification Engine）的Phase 2阶段，主要目标是实现记忆的可见性标签和敏感度分级功能，以增强系统的隐私保护能力。同时，确保系统保持高性能和轻量级的特性。

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 记忆的可见性标签
- 实现三级可见性标签：private（只有本人可见）、team（团队可见）、org（全组织可见）
- 默认为private，只有用户明确授权或记忆内容属于可分享类型，才升级为更高的可见性
- 在记忆元数据中存储可见性标签
- 检索时先过滤可见性，再进行语义匹配

#### 2.1.2 记忆的敏感度分级
- 实现三级敏感度分级：low（低敏感）、medium（中敏感）、high（高敏感）
- 基于内容自动判断敏感度
- 敏感度高的记忆不参与对外任务的检索
- 在记忆元数据中存储敏感度级别

#### 2.1.3 引用场景校验
- 实现引用场景校验模块，放在记忆检索结果返回之后、注入上下文之前
- 根据当前任务类型判断是否适合使用特定记忆
- 规则层覆盖大部分场景，如"task_type=external_output时，过滤掉sensitivity_level=high的记忆"

### 2.2 非功能需求

#### 2.2.1 性能要求
- 系统响应时间不超过100ms
- 内存使用不超过512MB
- 支持每秒至少100次请求

#### 2.2.2 可靠性要求
- 系统可用性达到99.9%
- 数据丢失率低于0.01%

#### 2.2.3 安全性要求
- 敏感数据加密存储
- 访问控制严格按照可见性标签执行
- 审计日志记录所有操作

## 3. 设计约束

- 保持系统轻量级，避免引入过多依赖
- 确保向后兼容性，不破坏现有功能
- 遵循隐私保护最佳实践

## 4. 验收标准

- 记忆可见性标签功能正常工作
- 记忆敏感度分级功能正常工作
- 引用场景校验功能正常工作
- 系统性能符合要求
- 系统安全性符合要求
- 所有测试用例通过
