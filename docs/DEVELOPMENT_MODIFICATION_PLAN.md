## 3. 技术方案

### 3.1 修复 SemanticUtility 类

**问题**: SemanticUtility 缺少 encode_text 方法，导致向量索引更新失败

**解决方案**:
- 在 SemanticUtility 类中添加 encode_text 方法
- 实现基于现有 embedding_model 的文本编码功能
- 提供回退机制，当模型不可用时使用字符串相似度

**代码示例**:
```python
def encode_text(self, text):
    """编码文本为向量"""
    if not self.embedding_model:
        # 回退到简单的字符串编码
        return [ord(c) for c in text[:100]]
    
    try:
        embedding = self.embedding_model.encode(text)
        return embedding.tolist()
    except Exception as e:
        logger.error(f"Text encoding failed: {e}")
        # 回退机制
        return [ord(c) for c in text[:100]]
```

### 3.3 提升日语处理能力

**问题**: 日语处理能力不足，分类准确率低

**解决方案**:
- 增强日语语言检测
- 添加日语特定的分类规则
- 优化日语文本的预处理
- 考虑使用日语特定的模型

**代码示例**:
```python
def detect_language(self, text):
    """检测文本的语言"""
    try:
        import langdetect
        return langdetect.detect(text)
    except Exception as e:
        # 日语特定检测逻辑
        if any(0x3040 <= ord(c) <= 0x30FF for c in text):
            return 'ja'
        return 'en'
```

### 3.4 优化长文本处理

**问题**: 长文本处理速度慢，影响用户体验

**解决方案**:
- 实现分块处理机制
- 采用并行处理
- 优化内存使用
- 增加缓存机制

**代码示例**:
```python
def process_long_text(self, text, max_chunk_size=1000):
    """处理长文本"""
    chunks = [text[i:i+max_chunk_size] for i in range(0, len(text), max_chunk_size)]
    results = []
    
    # 并行处理
    with concurrent.futures.ThreadPoolExecutor() as executor:
        futures = [executor.submit(self.process_message, chunk) for chunk in chunks]
        for future in concurrent.futures.as_completed(futures):
            results.append(future.result())
    
    # 合并结果
    return self._merge_results(results)
```

3. **提升日语处理能力** - 增强国际化支持
