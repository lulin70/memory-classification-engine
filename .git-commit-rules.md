# 记忆引擎 Git Commit Message 规则

## Commit 类型规范

使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范

### 主要类型

- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构（既不是新功能也不是修复）
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变动

### 作用域（Scope）

使用以下作用域标识修改的模块：

- `memory`: 记忆核心功能
- `storage`: 存储层（Tier 2/3/4）
- `classification`: 分类引擎
- `retrieval`: 检索功能
- `cache`: 缓存系统
- `api`: API 接口
- `config`: 配置文件
- `docs`: 文档
- `test`: 测试代码

## 记忆引擎特定规则

### 记忆存储相关

```bash
# 新增记忆类型
feat(memory): add user_preference memory type
feat(storage): implement tier2 yaml storage

# 优化存储逻辑
perf(storage): optimize sqlite connection pool
perf(storage): improve vector search performance

# 修复存储 bug
fix(storage): fix sqlite thread safety issue
fix(storage): resolve memory leak in cache
```

### 记忆分类相关

```bash
# 新增分类规则
feat(classification): add camelcase naming rule
feat(classification): support multi-language patterns

# 优化分类器
perf(classification): improve rule matching accuracy
perf(classification): reduce LLM API calls

# 修复分类问题
fix(classification): correct tier assignment logic
fix(classification): handle edge cases in pattern matching
```

### 记忆检索相关

```bash
# 新增检索功能
feat(retrieval): add semantic search support
feat(retrieval): implement time-decay ranking

# 优化检索性能
perf(retrieval): speed up vector search by 50%
perf(cache): add LRU cache for frequent queries

# 修复检索问题
fix(retrieval): fix relevance scoring bug
fix(retrieval): handle empty result sets
```

### 缓存系统相关

```bash
# 新增缓存功能
feat(cache): add in-memory SQLite cache
feat(cache): implement smart cache invalidation

# 优化缓存性能
perf(cache): reduce cache miss rate
perf(cache): optimize cache size management

# 修复缓存问题
fix(cache): fix dict conversion error
fix(cache): resolve cache consistency issue
```

### 配置和工具相关

```bash
# 新增配置
feat(config): add tier storage configuration
feat(config): support environment variables

# 工具脚本
chore(tooling): add cleanup script
chore(tooling): update dependencies
```

### 文档相关

```bash
docs(readme): update installation instructions
docs(api): add API documentation
docs(guide): add integration guide
```

### 测试相关

```bash
test(unit): add unit tests for storage layer
test(integration): add integration tests
test(performance): add benchmark tests
```

## Commit Message 格式

### 基本格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 示例

**简单提交：**
```bash
feat(memory): add user preference memory type
```

**带正文的提交：**
```bash
fix(storage): fix sqlite thread safety issue

- Add check_same_thread=False to sqlite connections
- Implement connection health check
- Add WAL mode for better concurrency

Closes #123
```

**带 BREAKING CHANGE 的提交：**
```bash
perf(storage): optimize connection pool

Refactored connection pool to use thread-local storage

BREAKING CHANGE: Connection pool API changed from get_connection() to get_connection(thread_id)
```

## 特殊规则

### 记忆相关提交

所有记忆相关的提交必须包含 `memory` 或具体模块作用域：

✅ 正确：
```bash
feat(memory): add new memory type
fix(storage): fix tier3 storage bug
```

❌ 错误：
```bash
feat: add new feature  # 缺少作用域
```

### 性能优化提交

性能优化必须说明优化效果：

✅ 正确：
```bash
perf(retrieval): speed up search by 50%
perf(cache): reduce memory usage by 30%
```

❌ 错误：
```bash
perf: improve performance  # 太模糊
```

### Bug 修复提交

Bug 修复应该引用 Issue 编号：

✅ 正确：
```bash
fix(storage): fix thread safety issue

Closes #456
Fixes #789
```

## 自动化工具

### 使用 TRAE 生成 Commit Message

```bash
# 自动生成
trae git commit --generate

# 使用自定义规则
trae git commit --rules .git-commit-rules.md
```

### 使用 Git Hooks

创建 `.git/hooks/commit-msg`：

```bash
#!/bin/bash
commit_msg=$(cat "$1")
if ! echo "$commit_msg" | grep -qE "^(feat|fix|docs|style|refactor|perf|test|chore)\(.*\):"; then
    echo "❌ Commit message 不符合规范！"
    echo "请参考 .git-commit-rules.md"
    exit 1
fi
echo "✅ Commit message 格式正确"
```

## 最佳实践

1. **保持简洁** - Subject 不超过 50 字符
2. **使用祈使句** - "add" 而不是 "added" 或 "adds"
3. **首字母小写** - 除非是专有名词
4. **不要句号** - Subject 末尾不要句号
5. **详细说明** - Body 详细说明为什么改，不是怎么改
6. **引用 Issue** - 使用 Closes #123 或 Fixes #456

## 示例集合

### 新功能
```bash
feat(memory): add sentiment marker memory type

Add support for detecting and storing sentiment markers
in conversations. Uses LLM-based classification with
confidence scoring.

Closes #100
```

### Bug 修复
```bash
fix(cache): fix dict conversion error in SQLite cache

Set row_factory to sqlite3.Row before fetching results
to properly convert to dictionary.

Fixes #234
```

### 性能优化
```bash
perf(storage): optimize connection pool for multi-threading

- Add thread-local storage for connections
- Implement connection health check
- Reduce lock contention by 40%

Benchmark results:
- Before: 100 ops/sec
- After: 170 ops/sec
```

### 重构
```bash
refactor(classification): extract rule matching logic

Move rule matching to separate RuleMatcher class
for better testability and reusability.

No functional changes.
```

### 文档
```bash
docs(readme): add installation and usage guide

Include:
- Installation steps
- Configuration options
- Usage examples
- API reference
```
